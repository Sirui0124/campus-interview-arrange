# campus-interview-arrange：校招约面排面系统

从 POPO 在线表格（固定模板）读取面试官时间和候选人清单，自动执行排面策略，结果写回表格。

---

## 执行流程

### Step 1: 确认输入

向用户确认：
1. POPO 表格链接（必须是包含固定5个Tab的模板）
2. 是否有特殊说明（如只排某些岗位、排除某些面试官等）
3. 排面结果写入哪个 Tab（默认写到"约面结果"）

### Step 2: Dry Run

```bash
cd ~/skills/campus-interview-arrange && python3 scripts/run_schedule.py "<popo_url>" --dry-run
```

向用户展示排面结果摘要，等待确认再写入。

### Step 3: 写入

用户确认后：
```bash
cd ~/skills/campus-interview-arrange && python3 scripts/run_schedule.py "<popo_url>" --write-to="约面结果"
```

---

## 核心策略

1. **约束匹配**：面试轮次 + 岗位/方向 + 形式 + 城市 必须全部兼容
2. **优先级贪心**：按评分从高到低排面（0分视为中间值2.5）
3. **2V1 四模式**：A(主随机+副随机) / B(主随机+副固定) / C(主随机+副N选一) / D(均主面)
4. **场次上限**：每位面试官每天不超过设定场次（≤2/≤4/≤6/≤8）
5. **交替安排**：同一面试官同天安排多人时，高低优先级交替
6. **海外优先**：海外候选人优先安排当地白天对应的时段

---

## 数据要求

POPO 表格必须包含以下 Tab（按 title 模糊匹配）：
- "面试官时间收集" — 面试官属性（轮次/岗位/2V1规则等）
- "面试官时间" — 面试官可用时段（日期+上午/下午/晚上）
- "候选人清单" — 待排面候选人列表
- "约面结果" — 输出目标

---

## 注意事项

- 已安排的候选人（安排状态="已安排"）自动跳过
- 面试官时间已被占用的 slot 不会重复分配
- 未排上的候选人会列出原因（无匹配面试官/时间不足）
